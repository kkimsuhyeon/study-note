# 코테 로드맵 — 프로그래머스 (진행 상황 추적)

> **목표**: 코테 가벼운 회사(중견·스타트업) 통과. 기준선 = **Lv.2를 시간 안에 안정적으로 푸는 수준**.
> **언어**: Java. **플랫폼**: 프로그래머스 "코딩테스트 연습" (백준 종료 이후 여기만 사용).
> **속도**: 하루 1문제로 충분. Spring 고급편이 메인, DSA는 지속 가능성이 우선.
>
> ⭐ = 그 유형의 **정석 문제** (시간 없으면 ⭐만 풀어도 감은 잡힌다)
> 푼 문제는 `- [x]`로 체크 → 진행 상황이 그대로 보인다.

---

## 진행 방식 (3줄)

1. **한 유형 몰아서 끝내고** 다음으로. 매일 섞으면 "반씩만 아는" 상태가 된다.
2. **30~40분 고민 → 안 되면 풀이 확인 → 3일 뒤 백지에서 다시 풀기.** 다시 풀 수 있어야 "완료".
3. 풀고 나면 **[신호 → 도구](./data-structure-selection.md)** 문서에 한 줄 추가. *풀이가 아니라 신호를 남긴다.*

문제 읽고 코드 짜기 전 **30초 루틴**: ① N이 얼마? → 허용 복잡도 역산 ② 내가 반복할 연산이 뭐지? ③ 그 연산을 빠르게 해주는 구조는? → [Big-O와 입력 크기](./big-o-and-input-size.md)

## STEP 1 — 워밍업: 해시 / 문자열 (Lv.1)

**목표**: 해시 감각 + 프로그래머스 환경 적응. (`import java.util.*;` 맨 위에, `package` 선언 금지, 클래스명 `Solution` 유지)

- [x] **완주하지 못한 선수** ⭐ — 해시 카운팅 (`merge`) · *2026-07-14 통과* → [케이스 정리](./data-structure-selection.md)
- [x] **폰켓몬** ⭐ — HashSet (중복 제거 / "종류가 몇 개") · *2026-07-14 통과*
- [ ] 두 개 뽑아서 더하기 ← **다음 문제**
- [ ] 문자열 내 p와 y의 개수
- [ ] 서울에서 김서방 찾기

> 💡 STEP 1의 갈림길: **중복이 의미를 갖나?** → 예: `Map`(개수) / 아니오: `Set`(존재). 완주하지 못한 선수는 동명이인 때문에 Map, 폰켓몬은 "종류"라서 Set. **두 문제로 양쪽을 다 겪음(2026-07-14).**

---

## STEP 2 — 스택 / 큐 / 힙

**목표**: "순서가 의미를 갖는" 문제 감각. Java는 스택·큐·덱 전부 `ArrayDeque` 하나로.

- [ ] 같은 숫자는 싫어 (Lv.1) — 스택 감각
- [ ] **크레인 인형뽑기 게임** (Lv.1) ⭐ — 스택
- [ ] **올바른 괄호** (Lv.2) ⭐ — 스택의 정석
- [ ] 기능개발 (Lv.2) — 큐
- [ ] 프로세스 (Lv.2) — 우선순위큐
- [ ] **더 맵게** (Lv.2) ⭐ — 힙 (`PriorityQueue`)

> 💡 정렬 vs 힙 갈림길: **한 번 정렬하고 끝?** → `sort` / **넣고 빼기를 반복?** → `PriorityQueue`

---

## STEP 3 — 정렬 / 이분탐색

- [ ] K번째수 (Lv.1) — 정렬 기본
- [ ] **가장 큰 수** (Lv.2) ⭐ — 정렬 + 커스텀 `Comparator`
- [ ] H-Index (Lv.2) — 정렬
- [ ] **입국심사** (Lv.3) ⭐ — 파라메트릭 서치 (이분탐색)

> 💡 파라메트릭 신호: **"조건을 만족하는 최대/최소값"** + 답의 범위가 정해져 있음 → 답을 이분탐색.

---

## STEP 4 — DFS / BFS ⭐⭐ (여기에 시간 제일 많이)

**코테 승부처.** 가벼운 코테에서 그나마 어려운 게 나온다면 여기.

- [ ] **타겟 넘버** (Lv.2) ⭐ — DFS / 백트래킹
- [ ] **게임 맵 최단거리** (Lv.2) ⭐ — BFS 최단거리
- [ ] **네트워크** (Lv.3) ⭐ — 연결 요소 개수
- [ ] 단어 변환 (Lv.3) — BFS
- [ ] 여행경로 (Lv.3) — DFS

> 💡 **"최단"이면 무조건 BFS.** DFS도 경로를 찾긴 하지만 최단이라는 보장이 없다. 예외가 거의 없으니 외울 것.
> 💡 2차원 격자도 그래프다 (상하좌우 = 간선).
